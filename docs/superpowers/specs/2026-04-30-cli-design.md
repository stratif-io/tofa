# rotp CLI — Design Spec

## Goal

Add a full-featured command-line interface to rotp so it can be used both as a TUI (default, no arguments) and as a scriptable CLI. Every command is covered by integration tests. The crate ships with shell completions and a GitHub-ready README.

## Architecture

`rotp/src/cli.rs` is replaced by a proper `rotp/src/cli/` module:

```
rotp/src/cli/
  mod.rs          — Clap root (Cli struct, dispatch)
  commands/
    init.rs
    destroy.rs
    list.rs
    code.rs
    add.rs
    remove.rs
    rename.rs
    qr.rs
    rekey.rs
    completions.rs
    export.rs
    import.rs
```

Each command file contains one public `run(args, vault_path)` function. No command touches the TUI. The TUI is launched only when `rotp` is invoked with zero subcommand arguments.

`rotp-core` is unchanged — all business logic already lives there.

## Vault resolution

Every command resolves the vault path in this order (highest priority first):

1. `--vault <path>` flag
2. `ROTP_VAULT` environment variable
3. `~/.config/rotp/vault.enc` (default)

## Passphrase resolution

Every command that needs the passphrase uses this order:

1. `ROTP_PASSPHRASE` environment variable — accepted with a stderr warning:
   `⚠  Passphrase read from ROTP_PASSPHRASE. Avoid this in production.`
2. Interactive prompt (hidden input, no echo)

## Commands

### `rotp` (no subcommand)
Launches the TUI. Unchanged from current behaviour.

---

### `rotp init [--vault <path>]`
Creates a new encrypted vault at the resolved path.
- Prompts for passphrase + confirmation.
- Fails with a clear error if the vault already exists:
  `Vault already exists at <path>. Use 'rotp rekey' to change the passphrase.`
- Creates parent directories if needed.

---

### `rotp destroy`
Permanently deletes the vault file.
- Prompts for passphrase to authenticate.
- Shows confirmation prompt: `Destroy vault at <path>? [y/N]`
- Exits with error if passphrase is wrong.

---

### `rotp list [--codes]`
Lists all accounts in the vault.

Without `--codes`:
```
GitHub:carlo
Authelia:carlo
OVH:mq31756
```

With `--codes`:
```
GitHub:carlo       480 152   25s
Authelia:carlo     391 047   25s
OVH:mq31756        112 834   25s
```

Exit code 0 even if vault is empty (prints nothing).

---

### `rotp code <name> [--raw] [--copy] [--watch]`
Displays the current TOTP code for the account matching `<name>`.

**Name matching:** case-insensitive substring match against the stored name.
- Exactly one match → proceed.
- Zero matches → error: `No account matching "<name>".`
- Multiple matches → error listing all matches + hint to be more specific.

**Flags:**
- `--raw` — prints bare digits with no space (`480152`). Suitable for `$(rotp code github --raw)`.
- `--copy` — copies to clipboard, prints confirmation to stderr: `Copied to clipboard.`
- `--watch` — refreshes in place every second:
  ```
  GitHub:carlo   480 152   ████████████░░░░░░░░   19s
  ```
  `Ctrl+C` to exit. Incompatible with `--raw` and `--copy`.

Default stdout output: `480 152`

---

### `rotp add [--name <name>] [--secret <base32>] [--uri <otpauth-uri>] [--qr <path>]`
Adds one account to the vault. Exactly one of `--secret`, `--uri`, or `--qr` must be provided.

- `--secret` requires `--name`.
- `--uri` parses issuer/account from the URI; `--name` overrides.
- `--qr` scans a PNG/JPEG; if the QR is a migration URI, all embedded accounts are imported (bulk import path).

After adding, confirms:
```
Added GitHub:carlo
Current code: 480 152  (25s)
```

---

### `rotp remove <name>`
Removes the matching account.
- Same matching rules as `rotp code`.
- Confirmation prompt: `Remove "GitHub:carlo"? [y/N]`

---

### `rotp rename <name> <new-name>`
Renames an account without touching the secret.
- Same matching rules as `rotp code` for `<name>`.
- `<new-name>` must be non-empty and not already taken.
- Prints: `Renamed "GitHub:carlo" → "GitHub:carlo-work"`

---

### `rotp qr <name|--all> [--output <path>]`
Exports a QR code.

- `rotp qr github` — displays the QR in the terminal using unicode block characters.
- `rotp qr github --output export.png` — saves as PNG.
- `rotp qr --all` — generates a Google Authenticator migration QR for all accounts.
- `rotp qr --all --output backup.png` — saves migration QR as PNG.

`<name>` and `--all` are mutually exclusive.

---

### `rotp rekey`
Changes the vault passphrase.
- Prompts for current passphrase (auth).
- Prompts for new passphrase + confirmation.
- Re-encrypts the vault with the new key.
- Prints: `Passphrase updated.`

---

### `rotp completions <bash|zsh|fish>`
Prints shell completions to stdout.

```bash
rotp completions bash >> ~/.bashrc
rotp completions zsh  > ~/.zfunc/_rotp
rotp completions fish > ~/.config/fish/completions/rotp.fish
```

Generated by Clap's built-in `clap_complete` crate.

---

### `rotp export [--output <path>]`
Exports all accounts as a plain-text JSON file.

Always prints to stderr before writing:
```
⚠  This file contains your secrets in plain text.
   Store it securely and delete it after use.
```

- Without `--output`: writes to `rotp-export-YYYY-MM-DD.json` in the current directory.
- With `--output <path>`: writes to the given path.

JSON format:
```json
[
  { "name": "GitHub:carlo", "secret": "JBSWY3DPEHPK3PXP", "created_at": "2026-04-30" }
]
```

Prints on completion: `3 accounts exported to rotp-export-2026-04-30.json`

---

### `rotp import <file>`
Imports accounts from:
- A JSON file produced by `rotp export`
- A PNG/JPEG image containing a Google Authenticator migration QR

Skips duplicates (same name + secret) with a warning. Prints a summary:
```
Imported 3 accounts. Skipped 1 duplicate.
```

---

## Error handling

- All errors print to stderr.
- Exit code 1 on any error, 0 on success.
- Wrong passphrase: `Error: wrong passphrase.` (no details to avoid oracle attacks).
- Vault not found: `Error: no vault at <path>. Run 'rotp init' to create one.`

## Output rules

- Informational messages → stderr (so stdout stays clean for piping).
- Data (codes, list, JSON export) → stdout.
- `--raw` always produces machine-readable output (no colour, no formatting).

## Testing

Every command has integration tests in `rotp/tests/cli_*.rs` using `assert_cmd` + `tempfile`:

- `cli_init.rs` — happy path, already-exists error, bad passphrase on destroy
- `cli_list.rs` — empty vault, populated vault, `--codes`
- `cli_code.rs` — exact match, partial match, ambiguous match, `--raw`, `--copy`
- `cli_add.rs` — `--secret`, `--uri`, `--qr` standard, `--qr` migration bulk
- `cli_remove.rs` — confirmation flow, not-found error
- `cli_rename.rs` — happy path, not-found, duplicate name
- `cli_qr.rs` — terminal output, `--output` PNG, `--all`
- `cli_rekey.rs` — happy path, wrong current passphrase
- `cli_completions.rs` — output non-empty for each shell
- `cli_export.rs` — JSON structure, warning on stderr
- `cli_import.rs` — JSON round-trip, migration QR, duplicate skip

`assert_cmd` crate added to `rotp` dev-dependencies.

## Documentation

### Man page
Generated from Clap via `clap_mangen` in a `build.rs` script. Output: `target/man/rotp.1`. Installed to `~/.local/share/man/man1/rotp.1` by the user.

### README
`README.md` at repo root covers:
- What rotp is (one paragraph)
- Installation (`cargo install rotp`)
- Quick start (5-line example)
- Full command reference (table)
- TUI screenshot
- Contributing / licence

### Inline help
Every command has a `--help` description written in `about =` / `long_about =` in the Clap definitions. Example:
```
$ rotp code --help
Get the current TOTP code for an account.

Name matching is case-insensitive and partial: "git" matches "GitHub:carlo".

Usage: rotp code [OPTIONS] <NAME>
...
```
